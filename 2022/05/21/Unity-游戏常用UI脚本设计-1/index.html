(1)房间列表面板

其中每一个块都是一个Panel,可以通过设置文本,按钮为各个Panel的子节点来制作上述的面板.

对于房间列表当中的房间,要单独进行设计,这里就要用到Scroll View组件.

(a)Scroll View UI组件

Scroll View是实现列表的最重要部件.**包含了名为Scroll Rect的组件,能够实现区域的滑动,还可以设置滑动的方向,绑定滑动条.**这里可以参考前面的博客对该组件的介绍.

对于房间列表,其Scroll Rect组件的个属性设置如下:

注意,这里面的Content中的Room是后面我们会填充到列表项中的内容,目前暂时先将其设定成空物体或者Panel即可.同时,如果新建Scroll View的话会自动为我们创建水平和垂直滚动条,如果没有特别修改的话应该是自动绑定好的,只需要修改Visibility的属性即可.

(b) 为Scroll View的Content添加GridLayoutGroup组件和Content Size Filter组件

  • GridLayoutGroup会自动地去排列Content的子物体(也就是我们的room),其中Cell Size指定列表项尺寸;padding是与边框的间距,Spacing是列表项之间的间隔;
  • Content Size Filter会自动调节Content尺寸;当列表项增多时,会自动增大Content,反之亦然.

(2)房间面板预设

玩家创建房间或者加入房间之后,会通过房间面板看到房间的详细信息.当然,面板还要包括”退出房间”和”开始战斗”两个按钮,同时我们希望能够看到一些基本的信息,比如当前房间里有哪些玩家,哪些玩家已经准备就绪等,如果是创建房间的房主玩家,则左下角会出现一个开战按钮,而对于加入其他房间的玩家来说,这个按钮可以是”准备”.(这个按钮的内容需要发生变化,比如玩家点击准备之后这个按钮应该变为”取消准备”,有一些细节后续要进行处理)

(3)一个注意事项(关于Content无法水平扩展)!

如果发现步骤与之前一样,但是Content无法完成很好地横向扩展(或者是竖向扩展),那么注意检查一下Content的Rect Transform当中的Left和Right数值是否是0(可能会由于不小心拖动或修改参数而导致问题,此时可以尝试这种解决方案)

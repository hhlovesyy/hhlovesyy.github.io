六.Github工作流

Github的工作流可以总结如下:

  • Create a new Branch:创建新的分支
  • Make changes and add Commits:做出改变并且commit
  • Open a Pull Request:开启一个pull request
  • Review: 预览
  • Deploy: 部署
  • Merge: 合并

接下来具体做出介绍:

1.创建新的分支

请注意:

  • 新的分支往往是要做一些实验性的工作所需要的,不要在这种情况下直接动master的代码:
  • 如果没有特殊情况,新的分支往往都是从master衍生出来的
  • 在团队合作的时候,新的分支起的名字要更好理解,方便团队更好的合作.

2.做出改变并commit

在创建一个新的分支之后,就可以修改里面的内容了.每当完成一件事的时候,可以add+commit.

  • 每个commit应该包括有用的信息,因为commit可以提供结点,使得出现问题的时候可以回滚,同时也方便其他合作者看到仓库的变化.

3.Open a pull request

这部分是Github的核心.一个pull request表示你改变了某些内容,方便其他人查看,并且选择是否要merge你的代码到他们的分支上.


4.review&deploy&merge

当一个Pull Request被发出时，它可以被任何对分支有适当访问权限的人审查。 这是对更改进行良好讨论和审查的地方。

如果您收到反馈可以继续改进您的更改，可以使用新的commit来添加更改，使进一步的审查成为可能.

当检查完pull request并发现没有太大的问题时，就该进行最后的测试了。 GitHub允许你在与主分支合并之前从一个分支进行部署，以便在生产中进行最终测试。

如果出现任何问题，您可以通过再次将主分支部署到生产中来撤消更改!

如果都没有问题的话,就可以和master分支合并了.
